# Benchmark Results

The numeric source of truth is [`data/benchmark-results-esp32s3.json`](../data/benchmark-results-esp32s3.json), generated by:

```bash
make update-benchmark-baseline
```

CI diffs every run against this baseline using `tools/benchmark_regression.py`. Only deterministic fields are compared (I/O counts, simulated latency, write amplification); wall-clock times are excluded.

---

## Key findings

All numbers below are from `data/benchmark-results-esp32s3.json`, generated with `data/calibration-esp32s3.json` (ESP32S3, QSPI flash). Simulated latency values are predicted on-device ms directly — no scaling needed.

**Enqueue** is dominated by rotation cost. At 64B payload (~46 records/segment) only ~2% of enqueues rotate; at 2048B every enqueue rotates. Simulated max latency is 79.369ms at all payload sizes — a single rotation. `readAt = 0` on the enqueue hot path.

**Peek+pop** costs one `readAt` (peek) + one `writeAt` (POP tombstone) per record. Simulated p99 = 25.432ms; max = 87.186ms (rotation on the pop path, rare — ~5 rotations per 1000 pops). `read_bpp` equals payload size exactly.

**Outbox offline submit** measures the failed-send path: transport returns `RetryLater`, falling through to durable enqueue. Write amplification is slightly higher than raw enqueue due to HTTP envelope framing (256B: 1.249× vs 1.110×).

**Mount** scales linearly with pre-loaded record count: empty = 15.634ms, 100 records = 2086.070ms, 1000 records = 19501.758ms. Each record requires one `readAt` to replay. Clean mount is read-only: `writeAt = 0`, `writeFile = 0`, `remove = 0`.

**Idle compaction** (burst=500, cycles=3, pop=90%): `hot_compactions = 0` confirms the clean-storage invariant — `compactIdle` between cycles eliminates dead data before the next burst, so the write path never stalls. Worst single step: 649.994ms at 256B payload, 1175.915ms at 492B.

## Strict-mode invariants

`--strict` exits 1 on any violation. The following must always hold:

- `enqueue`: `writeAt == N`, `readAt == 0`, `remove == 0`
- `peek_pop`: `writeAt == N`, `readAt == N`, `read_bpp == payloadBytes`, `remove == 0`
- `outbox_offline_submit`: `writeAt == N`, `read_bpp == 0`, `remove == 0`
- `mount`: `writeAt == 0`, `writeFile == 0`, `remove == 0`
- `idle_compaction`: `hot_compactions == 0`, `cap_exhausted == 0`
