# Benchmark Results

The numeric source of truth is [`docs/benchmark-results.json`](benchmark-results.json), generated by:

```bash
./build/pqueue-benchmark --json --strict --repeat 5 > docs/benchmark-results.json
```

CI diffs every run against this baseline using `tools/benchmark_regression.py`. Only deterministic fields are compared (I/O counts, simulated latency, write amplification); wall-clock times are excluded.

---

## Key findings

All numbers below are from `docs/benchmark-results.json`. Simulated latency predicts on-device ms at ×100 (ESP32S3, QSPI flash, default calibration).

**Enqueue** is dominated by rotation cost. At 64B payload (~46 records/segment) only ~2% of enqueues rotate; at 2048B every enqueue rotates. Simulated max latency is 3.934ms (×100 = ~394ms on device) at all payload sizes — a single rotation. `readAt = 0` on the enqueue hot path.

**Peek+pop** costs one `readAt` (peek) + one `writeAt` (POP tombstone) per record. Simulated p99 = 0.483ms; max = 4.279ms (rotation on the pop path, rare — ~5 rotations per 1000 pops). `read_bpp` equals payload size exactly.

**Outbox offline submit** measures the failed-send path: transport returns `RetryLater`, falling through to durable enqueue. Write amplification is slightly higher than raw enqueue due to HTTP envelope framing (256B: 1.249× vs 1.110×).

**Mount** scales linearly with pre-loaded record count: empty = 2.070ms sim, 100 records = 75.550ms, 1000 records = 729.638ms (×100 = ~73s on device). Each record requires one `readAt` to replay. Clean mount is read-only: `writeAt = 0`, `writeFile = 0`, `remove = 0`.

**Idle compaction** (burst=500, cycles=3, pop=90%): `hot_compactions = 0` confirms the clean-storage invariant — `compactIdle` between cycles eliminates dead data before the next burst, so the write path never stalls. Worst single step: 27.535ms sim at 256B payload, 48.533ms at 492B (×100 = ~2.8s and ~4.9s on device). The 492B result matches the on-device calibration run exactly.

## Strict-mode invariants

`--strict` exits 1 on any violation. The following must always hold:

- `enqueue`: `writeAt == N`, `readAt == 0`, `remove == 0`
- `peek_pop`: `writeAt == N`, `readAt == N`, `read_bpp == payloadBytes`, `remove == 0`
- `outbox_offline_submit`: `writeAt == N`, `read_bpp == 0`, `remove == 0`
- `mount`: `writeAt == 0`, `writeFile == 0`, `remove == 0`
- `idle_compaction`: `hot_compactions == 0`, `cap_exhausted == 0`
